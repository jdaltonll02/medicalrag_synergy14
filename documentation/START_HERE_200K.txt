TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: "Cannot connect to Elasticsearch"
Solution: Start Elasticsearch
  $ docker-compose -f docker/compose.yml up -d

Problem: "Index already exists"
Solution: Use --delete-existing flag or run delete script first
  $ python scripts/delete_index.py --config configs/pipeline_config.yaml --confirm

Problem: "Sampling takes too long" (>30 min)
Solution: This is normal! 33M documents = 15-25 minutes. Be patient.

Problem: Verification fails
Solution: Check individual components
  1. Check corpus file: ls -lh /data/.../pubmed_corpus_200k.jsonl
  2. Check ES: curl http://localhost:9200/medical_docs/_count
  3. Run verify script: python scripts/verify_setup.py --config configs/pipeline_config.yaml

See SETUP_200K_CORPUS.md → "Troubleshooting" for more details
