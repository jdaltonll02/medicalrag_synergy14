IMPLEMENTATION APPROACH
═════════════════════════════════════════════════════════════════════════════

Sampling Strategy:
  • Round 1: First 16.5M documents (first temporal snapshot)
  • Round 2: Second 16.5M documents (later temporal snapshot)
  • Distribution: 100K each for balanced representation
  • Reproducibility: Fixed random seed (default: 42)
  • Method: Streaming single-pass through corpus

Indexing Strategy:
  • Index Mappings: Optimized for medical text search
  • Analyzers: English language analyzer for title/abstract
  • Metadata: Snapshot round, date, source, ingestion time
  • Bulk Operations: Efficient 1000-document batches
  • Progress Tracking: Real-time updates every 10K docs

Verification Strategy:
  • File-level: Document count, structure, required fields
  • Index-level: Connectivity, statistics, document distribution
  • Data-level: Sample document inspection with field verification
  • Pass/Fail: Clear status with diagnostic information

TROUBLESHOOTING QUICK REFERENCE
═════════════════════════════════════════════════════════════════════════════

Elasticsearch Connection Failed:
  → Start: docker-compose -f docker/compose.yml up -d
  → Or manually start from: /data/.../elasticsearch/

Index Already Exists:
  → Use --delete-existing flag on reingest script
  → Or run delete_index.py first

Sampling Too Slow:
  → Normal for 33M documents (takes 15-25 min)
  → Progress is logged every 1M documents
  → Be patient, do not interrupt

Out of Memory:
  → Scripts use streaming (memory-efficient)
  → Check disk space: df -h
  → Reduce sample size if absolutely necessary

Verification Fails:
  → Run individual components:
  →   1. Check corpus: ls -lh pubmed_corpus_200k.jsonl
  →   2. Check ES: curl http://localhost:9200/medical_docs/_count
  →   3. Run verify script
